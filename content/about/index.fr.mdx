Je suis également un fervent défenseur de l'Open Source et j'aime contribuer à des projets qui me touche dans mes projets.

### Mes contributions Open Source

Node.js est un runtime Javascript que j'utilise depuis quelques années maintenant. J'ai eu l'occasion de contribuer à ce projet. Par example j'ai contribué à la traduction du site web de Node.js en français, ce qui a été une expérience enrichissante. J'ai ensuite travailé sur l'initative "website-redesign" de Node.js, qui vise à moderniser le site web de Node.js.
J'ai eu l'occasion de participer au "Nodejs collab summit" en novembre 2024 à Dublin, où j'ai pu rencontrer d'autres contributeurs et échanger sur les meilleures pratiques de développement. C'était une expérience incroyable de pouvoir discuter avec des personnes passionnées par le même sujet que moi.
Après celui-ci j'ai rejoins la toute fraiche initative "userland-migration" qui vise à aider les utilisateurs de Node.js à migrer vers la dernière version de Node.js.

L'écosystème JavaScript est puissant grace à ces nombreux packages mais le npm registry est un peu vieillissant. Lorsque deno à ouver au public son registre de package, j'ai décidé de contribuer par example UX, UI et quelque petit foncitonalité à ce registre. C'est [JSR](https://jsr.io), pour les curieux ou juste ceux qui veulent en savoir plus.

Tester des composant react peut être ennuyeux et compliqué quand on utilise Jest, c'est pourquoi j'ai essayé d'utiliser [`node:test` aka node test runner](https://nodejs.org/api/test.html) pour tester mes composants React. Mais il faillais un moyen d'importer les fichier `tsx`, `png`, `css` ... C'est comme cela que j'ai découvert [`nodejs-loaders/nodejs-loader`](https://github.com/nodejs-loaders/nodejs-loader) un package qui contient plusieur ESM loader pour Node.js initialement ecrit par [Jacob Smith](https://github.com/JakobJingleheimer). Et j'y ai contribué afin de le rendre meilleur. C'est comme cela que je suis devnus co-mainteneur de ce package.

Si vous voulez en savoir plus sur mes contributions Open Source, n'hésitez pas à consulter mon [profil GitHub](https://github.com/AugustinMauroy) ou alors d'allez sur la page [projets](/projects) de ce site.

## Mes outils (stack)

J'utilise principalement Node.js et React pour mes projets. J'aime aussi utiliser TypeScript pour ajouter une couche de sécurité à mon code. React tous seul peut être compliqué pour crée une app complète, c'est pourquoi j'utilise aussi Next.js pour créer des applications web. J'aime aussi utiliser Tailwind CSS pour styliser mes applications, car il me permet de créer des interfaces utilisateur élégantes et réactives rapidement. Et pour le stock de données, j'utilise souvent PostgreSQL avec Prisma.

Pour un developpement solide, j'utilise [BiomeJS](https://biomejs.dev) pour lint et formatter mon code. Et github pour héberger mes projets et collaborer avec d'autres développeurs. J'utilise aussi GitHub Actions pour automatiser mes workflows de développement, comme les tests et le déploiement de mes applications...
