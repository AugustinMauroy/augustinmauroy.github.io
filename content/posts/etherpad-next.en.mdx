## What is Etherpad-lite?

Etherpad is a highly customizable Open Source online editor providing collaborative editing in real-time. It is a web-based editor providing collaborative editing in real-time. It is a web-based editor that allows multiple people to write simultaneously in the same document.

## How it's actually working?

Etherpad-lite is a Node.js application that uses the `express` framework. It uses the `socket.io` library to handle real-time communication between clients and the server. The front-end is built with `jQuery`.

## Which technologies are used?

- [Next.js](https://nextjs.org/) as meta-framework it's allow us to do back-end and front-end in the same framework.
- [Tailwindcss](https://tailwindcss.com/) as css library. It's a utility-first css framework.
- [TypeScript](https://www.typescriptlang.org/) as language. It's a superset of JavaScript that compiles to plain JavaScript. It's allow us to have a readable and maintainable codebase.
- [next-ws](https://github.com/apteryxxyz/next-ws) it's a patch for next.js to allow us to use websockets on [route handler](https://nextjs.org/docs/app/building-your-application/routing/route-handlers). It's awesome because we didn't have to use express or socket.io. Thanks to [@apteryxxyz](https://github.com/apteryxxyz)
- [Prisma](https://www.prisma.io/) as ORM. It's allow us to have a better database management and a better codebase. Also it's create typescript types for us.

If you want to have better understanding of the project you can go to the [technical documentation](https://github.com/ether/etherpad-next/blob/main/docs/Techincal.md)

It's important to reduce breaking changes bettween etherpad-lite and etherpad-next.

We will try to keep the backward compatibility with etherpad-lite. But it's not a priority. We will focus on the new features and the new design.

### What I want to keep backward compatibility?

- Multiple db support for the project. But now Prisma isn't supporting multiple db. So we will have to find a solution. And for moment we only support MySQL.
- Plugins support. We will try to keep the same plugin system but we didn't know if it's possible.
- Moodle integration. Moodle is a popular LMS. We will try to keep the same integration but we didn't know if it's possible.
